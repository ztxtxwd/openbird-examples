# 定时任务设计

> 上游文档：[通用 Agent 架构设计](./2026-02-25-general-agent-architecture.md)

## 方案

利用飞书定时消息实现，不依赖外部定时系统。

## 已确定

### 飞书限制

飞书每人在一个会话里只能有一条待发送的定时消息，但可以编辑。

### 队列方案

定时消息的内容是一个按时间排序的任务队列：

```
⏰ 17:00 提醒老板开会
⏰ 17:30 回复李四报价单
⏰ 明天 9:00 给张三打电话
⏰ 周五 14:00 提交周报

定时发送时间 = 队列中第一项的时间
```

### 执行流程

1. 定时消息发出后，消息作为新信号触发 Agent
2. Agent 执行第一项任务
3. Agent 用剩余队列创建新的定时消息，发送时间 = 下一项的时间

### 插入新任务

新任务插入时按时间排序，如果新任务排在最前面，则编辑定时消息的发送时间。

## 待讨论

- 队列消息的格式规范（纯文本？结构化？方便 Agent 解析）
- 飞书定时消息内容的长度限制，以及大量任务时的处理
- 任务描述的粒度（"提醒老板开会"够吗？需要带上下文吗？）
- 跨天 / 跨周任务的时间表示
- 任务的编辑和删除（老板在工作台里说"取消明天的提醒"）
- 任务执行失败后的重试或上报
- 老板能否直接查看当前的定时任务队列
