# 工作台与"事"的设计

> 上游文档：[通用 Agent 架构设计](./2026-02-25-general-agent-architecture.md)

## 工作台

工作台是一个普通的飞书对话群，是老板和 Agent 的共享工作空间。

### 已确定

- 老板和 Agent 地位对等，双方都可以发起"事"、推进"事"、请求对方协助
- 工作台同时是 Agent 的配置界面——通过自然语言对话完成配置变更
- 工作台本身就是短期记忆——飞书即持久化层
- 定时任务队列也在工作台中（通过飞书定时消息实现）
- **消息不可变原则**——工作台里的消息一旦发出，Agent 不得编辑。所有更新通过追加新消息完成。原因：回复（如老板的"同意"）是对发出时刻内容的回应，编辑原消息会破坏回复与原文之间的语义关系，误导后续判断

## "事"

协作的最小单位。不是"消息"，不是"会话"，而是一件需要被处理和闭环的事务。

### 已确定

- 一条消息可能包含多件事，一件事可能横跨多个会话
- "事"的识别和拆解由 Agent 完成
- Agent 的核心职责标准：事事有回应
- 刺激与行动解耦——刺激从会话 A 进来，Agent 可能在会话 B 行动；也可能什么都不做，只更新记忆
- **表示形式：飞书话题（thread）**——每件"事"对应工作台群里的一个话题。新建一件事 = 发一条消息 + 创建话题；追加信息 = 回复已有话题；老板和 Agent 都在话题内推进和沟通

- "事"的生命周期——什么时候算闭环？谁来判断？
- 历史"事"的归档和清理
