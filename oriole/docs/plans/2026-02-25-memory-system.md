## 短期记忆

### 已确定

工作台（飞书话题群）本身就是短期记忆：
- 正在进行的"事"
- 跟老板的对话
- 定时任务队列

飞书即持久化层，不需要额外的短期记忆存储。

## 长期记忆

### 已确定

核心是**员工手册**——Agent 的行为准则文档。

- 由 Agent 从工作台对话中自动提炼，由老板确认生效
- 入职时手册为空，随培养过程生长
- 内容包括：行为准则、事务处理流程、老板偏好、优先级规则等
- 每次更新后 Agent 在工作台通知老板确认

员工手册的概念、运作机制和设计哲学详见 [Agent 产品哲学与培养范式](./2026-03-03-agent-philosophy-and-training-paradigm.md) 第 4 节。

### 读写机制

- Agent 启动时读取员工手册
- Agent 从老板的反馈中提炼规则，写入手册，并在工作台请老板确认
- 老板可以随时在工作台通过自然语言修改手册内容

## 待讨论

- 员工手册的存储位置（本地文件？飞书文档？数据库？）
- 员工手册的格式和结构
- 从对话到手册条目的提炼机制（由主 Agent 完成？还是独立的提炼流程？）
- 长期记忆和短期记忆（工作台）之间的信息流转——什么时候从工作台沉淀到手册？
- context window 中手册的注入方式（全量放入？按需检索？）
- 手册的上限和清理策略
- 是否需要手册之外的其他长期记忆文件（如联系人关系、老板画像等）
